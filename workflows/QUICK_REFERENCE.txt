================================================================================
TROUBLESHOOTING
================================================================================

PROBLEM: Output is blurry
FIX:     Increase steps from 10 to 12 (costs ~20% speed)
FIX:     Increase CFG from 2.0 to 2.5

PROBLEM: Visible noise/artifacts
FIX:     Regenerate with different seed (temporal anomaly)
FIX:     Reduce Turbo LoRA strength from 0.75 to 0.7

PROBLEM: Not following prompt
FIX:     Improve prompt specificity (more detailed description)
FIX:     Increase CFG from 2.0 to 3.0 (less photorealism, more control)
LIMIT:   Don't exceed CFG 4.0 (will degrade photorealism)

PROBLEM: CUDA out of memory
FIX:     Restart ComfyUI (clears VRAM cache)
FIX:     Reduce batch size from 2 to 1
FIX:     Close other GPU-intensive apps (Chrome, Blender, etc.)

PROBLEM: "Model not found" error
FIX:     Verify file exists: ComfyUI/models/diffusion_models/<filename>
FIX:     Re-run setup script: .\setup_flux_kria_secure.ps1

================================================================================
NODE ARCHITECTURE
================================================================================

Core Nodes Used: 9 (100% standard ComfyUI, zero experimental)

1. UNETLoader          → Load Flux Kria FP8 model
2. DualCLIPLoader      → Load CLIP-L + T5-XXL text encoders
3. VAELoader           → Load VAE encoder/decoder
4. CLIPTextEncode      → Encode positive prompt (photography keywords)
5. CLIPTextEncode      → Encode negative prompt (quality safeguards)
6. LoraLoader          → Apply Turbo LoRA @ 0.75 strength
7. EmptyLatentImage    → Initialize 1024x1024 latent space
8. KSampler            → Generate 10 steps, Euler, Simple, CFG 2.0
9. VAEDecode           → Convert latent to image
10. SaveImage          → Save with "production_photo_" prefix

NO EXPERIMENTAL NODES. NO THIRD-PARTY MODULES. 100% STABLE.
