Status: INFORMATIONAL - Design issue, not critical

Highly Redundant Packages (installed by 3+ nodes):
  1. transformers: 6 nodes
     - ComfyUI (base), ComfyUI-Impact-Pack, ComfyUI-IPAdapter-Flux,
       ComfyUI-Manager, was-node-suite-comfyui, x-flux-comfyui
     - Total installations: 6

  2. einops: 4 nodes
     - ComfyUI (base), ComfyUI-IPAdapter-Flux, comfyui_controlnet_aux, x-flux-comfyui

  3. opencv-python: 4 nodes
     - ComfyUI-Inspire-Pack, ComfyUI-RMBG, comfyui_controlnet_aux, x-flux-comfyui

  4. sentencepiece: 4 nodes
     - ComfyUI (base), ComfyUI-GGUF, ComfyUI-IPAdapter-Flux, x-flux-comfyui

  5. matplotlib: 4 nodes
     - ComfyUI-Impact-Pack, ComfyUI-Inspire-Pack, comfyui_controlnet_aux, was-node-suite-comfyui

  6. scikit-image: 3 nodes
     - ComfyUI-Impact-Pack, comfyui_controlnet_aux, was-node-suite-comfyui

  7. gitpython: 3 nodes
     - ComfyUI-Manager, was-node-suite-comfyui, x-flux-comfyui

  8. protobuf: 3 nodes
     - ComfyUI-GGUF, ComfyUI-IPAdapter-Flux, ComfyUI-RMBG

Problem Description:
  Each node specifies its own requirements.txt. Pip installs each dependency
  independently, even when the same package is needed by multiple nodes.
  This creates:
    - Longer installation times
    - Larger disk usage (multiple copies of same code)
    - Harder dependency management
    - Risk of version mismatches across nodes

Root Cause:
  ComfyUI custom_nodes architecture allows independent package management.
  No centralized dependency consolidation mechanism.

Impact on RTX 3090 Setup:
  - Installation time: ~15-20 minutes (packages downloaded/installed multiple times)
  - Disk space: Package duplication adds ~500MB-1GB
  - Maintenance: Updating transformers requires validation across 6 nodes

Mitigation Options:
  OPTION A: Consolidate to top-level requirements.txt
    - Create D:\workspace\fluxdype\custom_nodes_requirements.txt
    - List all unique dependencies with relaxed versions
    - Remove redundant individual requirements.txt files
    - Install with: pip install -r custom_nodes_requirements.txt
    - Impact: Faster installation, easier maintenance
    - Risk: Requires testing all node combinations

  OPTION B: Use pip caching
    - Already enabled by default in pip
    - Minimizes re-downloads but doesn't reduce redundancy

  OPTION C: Accept current design
    - Flexibility: Each node manages own dependencies
    - Trade-off: Longer install times, more storage

Recommended Action:
  Document in setup scripts that redundant packages are expected.
  Consider consolidation if installation time becomes problematic.
